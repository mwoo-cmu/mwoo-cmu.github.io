Enemies

Enemy Primary and Secondary Units have an j deck. On the enemy turn, draw an Order card from their Order deck. Play cards from the selected enemy's j deck equal to their j number.
The Enemy Primary Unit also has a i deck. Whenever it is Wounded, play a card from their i deck.

Whenever an enemy Unit makes an attack, remove one die from the attack roll for each Wound or Injury it has.

Grid System

g: count the number of orthogonal spaces in the shortest path from one object to another.
p: The affected character can be moved 1 g away from (backwards or diagonally backwards, but not horizontal) the shoving character, then the shoving character may immediately move toward the affected character 1 g.
Pull: The affected character can be moved towards (forwards or diagonally forwards, but not horizontal) the pulling character. Alternatively, move the affected character along the shortest path to the pulling character, even if this would increase the Euclidean distance between the two.
x: Move up to 5 spaces, can move through allies, but not enemies; must end on an empty space. If Engaged, h instead.
h: Move up to 2 spaces, can move through allies and enemies; must end on an empty space.
t: Move to any other empty space in LOS.
u: Convert to h.
LOS: two grid spaces have Line of Sight as long as an uninterupted line can be drawn from any point within one grid space to any point within the other.
Cover: The LOS requirement for Cover applies similarly. If an interupted line from any point in the attacker's grid space to any point in the defender's grid space, this condition is met.

Locations

Each time you reshuffle your Order deck, draw an Event card and resolve its effects.
Mission token: A marker used for tracking missions. The specific interactions of this token are dependent on the Mission.
Priority Objective token: A marker used primarily by the Enemy deck to determine movement.

Stacking Supporting Units

If things are getting too cluttered, or there aren't enough minifigures, you may choose to stack Supporting Units together. The Unit Stack has the combined r and w of all of the stacked Supporting Units. A Unit Stack's dice pool for attack and defence increases by one for each extra character added to it. Each time the Unit Stack is Injured, remove one character from the stack.
